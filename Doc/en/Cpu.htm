- cpu -

Description

Cpu agent is used to monitor CPU Usage.

Configuration

  • setting directive syntax : 
    <setting name="" value="" />
  • generic settings

    name value
    alwaysgreen Possible values are "true" or "false". If true, it will always send green status. Default is false.
    limit Limit is a numeric value. This will limit the number of lines of the process list Default is 0 which means to list all processus
    psmode Possible values are "true" or "false". If true, it will report all the processus ordered by cpu usage. Default is true
    testname You can set the testname that will be sent to the monitoring server (column name in the bbdisplay) Default is "cpu"

  • rules directive syntax :

    <setting name="" rule="" alarmcolor="" comment=""/>

    If the name value is not one of the generic settings, then, cpu will understand that it is the cpu default rule.


    Attribute Value
    delay Delay is numeric value. It allows to create an internal counter before sending alarms about cpu usage. If delay is set to 4, it will wait 4 cpu runs before sending an alarm. If between 2 runs, the cpuusage come back to a value less than the warnlevel, then, the internal counter will be reset to 0. This permits to refine cpu usage detection. Default value is 3
    name Must be "default".
    paniclevel It must be a numeric value. It will warn red if the value is reached. Default is 95%
    warnlevel It must be a numeric value. It will warn yellow if the value is reached. Default is 90%
    uptime This option is used to monitor the uptime via the cpu report like the original Big Brother client. This option is not activated by default. To turn it on, you may specify the uptime setting with a time value corresponding to the delay you want to warn for a reboot which has been done recently. Unit is seconds. You can specify a unit after the numeric value : "m" for minutes, "h" for hours, "d" for days.

Examples

<cpu>
	<setting name="alwaysgreen" value="false" />
	<setting name="default" warnlevel="90" paniclevel="95" delay="3" />
</cpu>

Notes

  • Owner doesn't appear in the status report on Windows NT4

Procs

Description

Procs agent is used to monitor processus.

Configuration

  • setting directive syntax : 
    <setting name="" value="" />
  • generic settings

    name value
    testname You can set the testname that will be sent to the monitoring server (column name in the bbdisplay) Default is "procs"
  • rules directive syntax : 
    <setting name="" rule="" alarmcolor="" comment=""/>

    If the name value is not one of the generic settings, then, procs will understand that it is a procs custom rule. To create a custom procs rule, you may use the process name in the name attribute.


    Attribute Value
    alarmcolor You can set the alarm color you want to use if their is an alert. It should be set to red or yellow. Default is yellow.
    comment A simple string which appear in the report to add a small information about the processus monitored if its name is not easy to understand. This parameter is optionnal
    name name attribute should be the name of the process you want to monitor. This parameter is required
    rule Rule is composed of a numeric sign followed by a numeric value. Possibles signs are '-=', '+=', '-', '+' or '='. '-' means < and '+' means >. See the example below. This parameter is required

Examples

  • procs is configured to check that no drwtsn process are running ; it will warn red if one is present in memory. Also, it check that one instance of pageant.exe is always running ; it will warn yellow if its count is different than 1 
    <procs>
	<setting name="drwtsn" rule="-1" alarmcolor="red" />
	<setting name="pageant.exe" rule="=1" comment="Putty agent deamon" />
</procs>

Notes

  • proc name in the procs rules can be part of it and is case insensitive.
